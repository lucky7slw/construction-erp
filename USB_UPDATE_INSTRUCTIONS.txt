USB UPDATE SYSTEM INSTRUCTIONS
===============================

OVERVIEW:
---------
Updates are transferred via USB flash drive from your dev computer to the server.


ON DEV COMPUTER (Windows/Mac/Linux):
-------------------------------------
1. Edit prepare_update.sh:
   - Set USB_PATH to your USB mount point (e.g., /media/usb, /Volumes/USB, D:\)
   - Set the path to your application files

2. Prepare update package:
   ./prepare_update.sh 1.0.1

3. This creates on USB:
   - version.json (contains version number)
   - update.tar.gz (your application files)

4. Safely eject USB drive


ON SERVER (MacBook with Linux):
--------------------------------
1. Plug in USB drive

2. Check USB mount location:
   lsblk
   # Usually /media/usb or /media/username/USB_NAME

3. Edit auto_updater.py if needed:
   - Update USB_MOUNT_PATH to match your USB location

4. Go to Admin → System Updates tab in web UI

5. Click "Check for Updates"
   - It will detect the new version on USB

6. Click "Install Update Now"
   - Stops service
   - Copies files from USB
   - Installs update
   - Restarts service


AUTOMATIC UPDATES:
------------------
The system checks for USB updates at 3:00 AM daily.
Just leave the USB plugged in and it will auto-update.


FILE STRUCTURE ON USB:
----------------------
USB_DRIVE/
├── version.json       # {"version": "1.0.1"}
└── update.tar.gz      # Your packaged application


TROUBLESHOOTING:
----------------
- "USB drive not found": Check mount path with 'lsblk' or 'df -h'
- "No version.json found": Make sure files copied to USB correctly
- Check logs: tail -f update.log
